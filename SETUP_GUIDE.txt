PetKO - Setup Guide for Groupmates
====================================
This guide covers how to run, edit, and build both the web dashboard
and the mobile app on your own machine.


PREREQUISITES
-------------
Make sure you have these installed before starting:

  PHP 8.2+          https://www.php.net/downloads
  Composer          https://getcomposer.org
  Node.js (v18+)    https://nodejs.org
  Git               https://git-scm.com
  Expo Go (phone)   App Store / Play Store


STEP 1 - Clone the Repository
------------------------------
  git clone <your-repo-url>
  cd PetKO-Dashboard


STEP 2 - Set Up the Web Dashboard (Laravel + React)
----------------------------------------------------
  cd petko-dashboard

  composer install
  npm install
  cp .env.example .env
  php artisan key:generate
  php artisan migrate
  php artisan db:seed

Run the dashboard (open TWO separate terminals inside petko-dashboard):

  Terminal 1 - Frontend:
    npm run dev

  Terminal 2 - Backend:
    php artisan serve --host=0.0.0.0 --port=8000
    (use --host=0.0.0.0 so the mobile app can reach it)

Open your browser at: http://127.0.0.1:8000

Default login accounts:
  Admin  |  admin@petko.com  |  petko2026
  Staff  |  staff@petko.com  |  staff2026


STEP 3 - Set Up the Mobile App (Expo)
--------------------------------------
  cd petko-mobile
  npm install
  npx expo install expo-print expo-sharing expo-file-system expo-font

The mobile app connects to the backend via config.js.
You have two options:

  OPTION A - Use Railway (live server, needs internet):
    Edit petko-mobile/src/config.js:
      export const API_BASE = 'https://bipetkoweb-mobile-production.up.railway.app';
    Then just run: npx expo start
    No need to run Laravel locally.

  OPTION B - Use local server (same Wi-Fi, no internet needed):
    Find your IP:
      Windows -> open Command Prompt -> run: ipconfig
                 look for "IPv4 Address" under your Wi-Fi adapter
    Edit petko-mobile/src/config.js:
      export const API_BASE = 'http://YOUR_IP_HERE:8000';
    Make sure Laravel is running with --host=0.0.0.0
    Then run: npx expo start

Run the mobile app:
  npx expo start

Scan the QR code using the Expo Go app on your phone.

!! Your phone and PC must be on the SAME Wi-Fi network for Expo Go to work.
!! Mobile data also works if config.js points to Railway.


STEP 4 - Making Changes
------------------------
Web dashboard changes (React components):
  - Edit files in petko-dashboard/resources/js/components/
  - Changes appear instantly while npm run dev is running

Web dashboard changes (Laravel/PHP):
  - Edit files in petko-dashboard/app/Http/Controllers/
  - No restart needed, changes apply immediately

Mobile app changes:
  - Edit files in petko-mobile/src/screens/ or petko-mobile/src/components/
  - Changes appear instantly in Expo Go (just save the file)

After making changes, push to GitHub:
  git add .
  git commit -m "describe your change"
  git push

Railway will automatically redeploy the web dashboard after each push.


STEP 5 - Building the Android APK
-----------------------------------
When you want to distribute the app without Expo Go:

  npm install -g eas-cli
  eas login
  cd petko-mobile
  eas build --platform android --profile preview

This builds in the cloud (free tier, may queue).
Download the APK from the link provided and install on Android phones.

!! Make sure config.js points to Railway before building the APK.
!! After building, update config.js back to local IP if needed for development.


STEP 6 - Pushing Data to Railway Database
-------------------------------------------
If you need to reimport CSV data to the live Railway database:

  1. Edit petko-dashboard/.env temporarily:
       DB_CONNECTION=mysql
       DB_HOST=shinkansen.proxy.rlwy.net
       DB_PORT=36994
       DB_DATABASE=railway
       DB_USERNAME=root
       DB_PASSWORD=yQkRxOaWMGtkhFcuEGkjgVgnOpjpOwxD

  2. Run: php artisan db:seed --class=CsvImportSeeder

  3. Change .env back to: DB_CONNECTION=sqlite


COMMON ISSUES
-------------
"Network request failed" on login
  - Make sure Laravel is running with --host=0.0.0.0
  - Double-check the IP in config.js matches your machine's actual IPv4
  - Make sure your phone and PC are on the same Wi-Fi

"expo-asset cannot be found"
  cd petko-mobile
  npm install expo-asset

Blank screen or app crashes on start
  cd petko-mobile
  npx expo start --clear

Database errors on first run
  cd petko-dashboard
  php artisan migrate:fresh --seed

Duplicate import error in App.js
  Make sure MaterialCommunityIcons is only imported once at the top of App.js

NOTES
-----
- The web dashboard and mobile app share the same Laravel backend API.
- Only ONE person needs to run the Laravel server locally. Everyone else
  just updates config.js to point to that person's IP address.
- Admin: can manage users, view analytics, export reports.
- Staff: can use POS, view inventory, and view low stock alerts.
- All data is stored in Railway's MySQL database.
- The .env file is for local development only. Railway uses its own
  environment variables set in the Railway dashboard.
